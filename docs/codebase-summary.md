## File Details

### VoxType/App/VoxTypeApp.swift (101 lines)

Entry point. Two types:

- **`VoxTypeApp: App`** -- `@main` struct. Creates `SettingsStore` and `AppController` as `@StateObject`. Declares `Settings` scene with `SettingsView` and onboarding `Window` with `OnboardingView`.
- **`AppController: ObservableObject`** -- `@MainActor`. Owns all service instances. Receives `SettingsStore` via init, creates `HotkeyManager(settings:)`. Shows onboarding on first launch. Wires Combine bindings from `DictationManager` to `MenuBarController` and `HUDController`. Calls `TranscriptionService.prepareModel()` in background `Task` on init.

### VoxType/App/MenuBarController.swift (98 lines)

- **`MenuBarController`** -- `@MainActor`. Creates `NSStatusItem` with "mic" SF Symbol. Menu includes model status (disabled info item), Settings (Cmd+,), About (version 1.0.0), and Quit. `updateIcon(for:)` switches SF Symbol based on `DictationState`. Holds weak reference to `TranscriptionService` for model status display.

### VoxType/App/OnboardingController.swift (107 lines)

- **`OnboardingController: ObservableObject`** -- `@MainActor`. Manages 5-step onboarding (Welcome, Mic, Accessibility, Model Download, Try It).
  - Published: `step`, `canProceed`, `micPermission`, `accessibilityPermission`, `modelStatus`
  - Observes `TranscriptionService.$modelStatus` via Combine
  - `requestMicPermission()`: delegates to `AudioCaptureService`
  - `openAccessibilitySettings()`: uses `AXIsProcessTrustedWithOptions` + 1s polling timer
  - Sets `settings.hasCompletedOnboarding = true` on step 5 completion
- **`PermissionState`** -- enum (notDetermined, authorized, denied, restricted). Static factory methods `.mic()` and `.accessibility()` check current system state.

### VoxType/Features/DictationState.swift (14 lines)

- **`DictationState: Equatable`** -- Enum with associated values: `.idle`, `.listening`, `.transcribing`, `.done(text: String)`, `.error(message: String)`. Computed `isProcessing` returns true for listening/transcribing.

### VoxType/Features/DictationManager.swift (190 lines)

- **`DictationManager: ObservableObject`** -- `@MainActor`. Orchestrates full dictation lifecycle.
  - Published: `state`, `audioLevel`
  - `bindHotkey()`: subscribes to HotkeyManager keyPressed/keyReleased + silenceDetected (auto-stop when enabled)
  - `startDictation()`: precondition guards (model ready, mic permission), can interrupt in-flight transcription
  - `stopDictation()`: calls `audioService.stopRecording()`, skips if audio < 0.3s (4800 frames), spawns async transcription
  - `transcribeAndInsert(_:retryCount:)`: chains TranscriptionService -> TextInsertionService, logs latency via ContinuousClock. Retries once with 200ms backoff on failure. Provides suggested actions per error type.
  - `scheduleReset()`: 3-second Task.sleep then reset to idle

### VoxType/Services/AudioCaptureService.swift (204 lines)

- **`AudioCaptureService: ObservableObject`** -- Not @MainActor. Manages AVAudioEngine.
  - Target format: 16kHz mono Float32
  - `silenceDetected`: PassthroughSubject, fires when RMS < 0.02 for configurable duration (default 1.5s = 24000 frames at 16kHz)
  - `startRecording()`: creates engine, installs tap with `TapHandler` inner class for format conversion. Reads `silenceTimeout` from UserDefaults to configure silence frame limit.
  - `TapHandler`: @unchecked Sendable, calculates RMS level, converts via AVAudioConverter, appends to pcmBuffers, increments silence frame count
  - `stopRecording()`: removes tap, stops engine, calls `mergeBuffers()`
  - `mergeBuffers()`: concatenates all PCM buffers into single buffer
  - `hasMicPermission`: checks `AVCaptureDevice.authorizationStatus(for: .audio)`
  - `requestMicPermission()`: async wrapper around `AVCaptureDevice.requestAccess(for: .audio)`
- **`AudioError`** -- enum: formatError, notRecording

### VoxType/Services/TranscriptionService.swift (116 lines)

- **`TranscriptionService: ObservableObject`** -- `@MainActor`. Manages WhisperKit.
  - Model: `openai_whisper-large-v3-turbo`
  - `prepareModel()`: downloads and loads WhisperKit model
  - `transcribe(_:)`: writes PCM to temp WAV, calls `whisperKit.transcribe(audioPath:)`, deletes temp file
  - `writePCMTempoWAV()`: manual RIFF header + IEEE Float32 data
  - Published `modelStatus: ModelStatus` (notLoaded, downloading, loading, ready, failed)
  - `isReady`: computed, returns `modelStatus == .ready`
- **`TranscriptionError`** -- enum: modelNotReady, invalidAudio, emptyResult

### VoxType/Services/HotkeyManager.swift (216 lines)

- **`HotkeyManager: ObservableObject`** -- Global hotkey via CGEvent tap. Configurable key from SettingsStore.
  - Target: configurable (default keyCode 61 / Right Option). Reads from UserDefaults, supports modifier-only and combo keys.
  - `keyPressed` / `keyReleased`: PassthroughSubject<Void, Never>
  - `init(settings:)`: reads initial hotkey from UserDefaults, observes `UserDefaults.didChangeNotification` to reinstall tap on config change
  - `installEventTapOrPoll()`: if accessibility granted, installs tap; otherwise polls at 1s until granted
  - `handleEvent()`: filters for target keyCode + modifiers, tracks isKeyDown state, re-enables tap on timeout. Handles `flagsChanged` for modifier-only keys.
  - `isModifierPressed()`: checks CGEventFlags for modifier key state
  - `hasAccessibility`: checks AXIsProcessTrusted
  - `requestAccessibility()`: prompts user with system dialog
  - Requires Accessibility permission (AXIsProcessTrusted)

### VoxType/Services/TextInsertionService.swift (84 lines)

- **`TextInsertionService`** -- Clipboard paste and Cmd+V simulation.
  - `insertText(_:)`: saves clipboard, writes text, captures changeCount as baseline, 50ms delay, simulates Cmd+V, schedules restore after 500ms (only if changeCount unchanged -- avoids overwriting user copies)
  - `simulatePaste()`: CGEvent keyDown/keyUp for virtual key 0x09 with .maskCommand
  - `hasAccessibility`: checks AXIsProcessTrusted
  - Returns `InsertionResult`: success, clipboardOnly, failed(String)

### VoxType/Services/SettingsStore.swift (81 lines)

- **`SettingsStore: ObservableObject`** -- `@MainActor`. @AppStorage-backed settings persistence via UserDefaults.
  - General: `launchAtLogin` (SMAppService register/unregister), `silenceDetectionEnabled`, `silenceTimeout` (0.5-3.0s, default 1.5)
  - Hotkey: `hotkeyKeyCode` (default 61), `hotkeyModifiers` (default 0)
  - Advanced: `soundFeedback`, `debugLogging`
  - Onboarding: `hasCompletedOnboarding`
  - `hotkeyDisplayName`: computed, combines modifier symbols + key name
  - `updateLaunchAtLogin()`: registers/unregisters with SMAppService, reverts toggle on error
  - `keyCodeName(_:)`: maps CGKeyCode to human-readable name (Right Option, Right Command, etc.)
  - `modifierFlagNames(_:)`: maps CGEventFlags to symbols (Ctrl, Opt, Shift, Cmd)

### VoxType/UI/SettingsView.swift (117 lines)

- **`SettingsView`** -- TabView with three tabs: General, Hotkey, Advanced. Fixed width 450pt.
- **`GeneralSettingsView`** -- Launch at Login toggle, silence detection toggle + timeout slider (0.5-3.0s, 0.1 step).
- **`HotkeySettingsView`** -- Displays current hotkey name, "Record New" button opens inline `HotkeyRecorderView`. Cancel to dismiss.
- **`AdvancedSettingsView`** -- Sound feedback toggle, debug logging toggle.

### VoxType/UI/HotkeyRecorderView.swift (152 lines)

- **`HotkeyRecorderView: NSViewRepresentable`** -- SwiftUI bridge to RecorderNSView. Callbacks: `onRecorded(CGKeyCode, UInt)`, `onCancel()`.
- **`RecorderNSView: NSView`** -- Captures key input via NSEvent local monitors (keyDown + flagsChanged).
  - Supports modifier-only keys (Right Option, Caps Lock, etc.) via flagsChanged detection
  - Supports combo keys (Ctrl+Space, etc.) via keyDown + modifier extraction
  - Escape cancels recording
  - Blocks system shortcuts (Cmd+Q, Cmd+W, Cmd+H, Cmd+M, Cmd+Tab)
  - `isModifierKeyCode()`: identifies keyCode as modifier (codes 54-62)
  - `modifierFlagForKeyCode()`: maps keyCode to CGEventFlags modifier value

### VoxType/UI/OnboardingView.swift (178 lines)

- **`OnboardingView`** -- 5-step onboarding UI. Step indicator (5 dots), Back/Next navigation. Fixed 480x360pt.
  - Step 1 (Welcome): app introduction
  - Step 2 (Mic): permission badge + "Grant Access" button
  - Step 3 (Accessibility): permission badge + "Open System Settings" button, restart warning if denied
  - Step 4 (Model): model status display (waiting/loading/ready/failed)
  - Step 5 (Try It): usage instructions + hint about Settings (Cmd+,)
- **`permissionBadge(_:)`** -- ViewBuilder rendering PermissionState as colored label

### VoxType/UI/HUDView.swift (199 lines)

Five types in one file:

- **`HUDPanel: NSPanel`** -- Borderless, non-activating, floating level, clear background, all spaces. 260x52pt frame.
- **`HUDState`** -- `@MainActor @Observable`. Shared state object holding `state: DictationState` and `audioLevel: Float`. Injected into SwiftUI via `.environment(hudState)`.
- **`HUDController`** -- `@MainActor`. Manages panel lifecycle. `show(state:audioLevel:)` creates panel lazily, positions near menu bar center. Uses HUDState for SwiftUI content updates. `updateAudioLevel(_:)` re-renders during listening.
- **`HUDContentView: View`** -- SwiftUI. Icon + label + optional audio bar / result preview / error message. Uses `.ultraThinMaterial` background with rounded corners. Reads state from `@Environment(HUDState.self)`.
- **`AudioLevelBar: View`** -- Red progress bar proportional to RMS level (min 2pt width, 4pt height).

## Dependency Graph

```
VoxTypeApp
  └── AppController
        ├── SettingsStore (shared, injected into multiple components)
        ├── MenuBarController (weak ref to TranscriptionService for model status)
        ├── HUDController
        │     └── HUDPanel + HUDState + HUDContentView + AudioLevelBar
        ├── DictationManager
        │     ├── AudioCaptureService
        │     ├── TranscriptionService  (-> WhisperKit)
        │     ├── TextInsertionService
        │     └── HotkeyManager (reads SettingsStore via UserDefaults)
        └── OnboardingController (only on first launch)
              ├── AudioCaptureService (mic permission)
              └── TranscriptionService (model status)
```

All services are instantiated by `AppController` and injected into `DictationManager` via init parameters. `SettingsStore` is passed to `AppController`, `HotkeyManager`, and `OnboardingController`. `HotkeyManager` reads hotkey config via UserDefaults (not SettingsStore directly) to avoid @MainActor crossing. No service discovers another service -- all wiring is explicit.

## Key Protocols / Patterns

| Pattern | Usage |
|---|---|
| ObservableObject + @Published | State propagation (DictationManager, TranscriptionService, AudioCaptureService, SettingsStore, OnboardingController) |
| @Observable | HUD state (HUDState) |
| @AppStorage | Settings persistence (SettingsStore) |
| PassthroughSubject | Event signaling (HotkeyManager key events, AudioCaptureService silence detection) |
| Combine sink | Binding services to UI (AppController.wireUI) |
| UserDefaults.didChangeNotification | HotkeyManager reacts to settings changes |
| @MainActor | UI-bound classes (AppController, DictationManager, TranscriptionService, HUDController, SettingsStore, OnboardingController) |
| DispatchQueue | Audio buffer thread safety (AudioCaptureService.bufferQueue) |
| async/await | WhisperKit transcription, text insertion delays |
| SMAppService | Launch at login (SettingsStore) |
| NSViewRepresentable | HotkeyRecorderView bridges AppKit NSEvent monitoring to SwiftUI |
| Delegation via init injection | DictationManager receives all services as constructor parameters |
